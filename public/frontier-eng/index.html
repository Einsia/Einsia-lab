Abstract
We introduce Frontier-Engineering Bench, an open benchmark designed to evaluate frontier AI models on real-world engineering problems spanning 12+ technical domains — from aerospace and chip design to quantum circuits and computational fluid dynamics. Unlike existing benchmarks that test factual recall or code generation in isolation, Frontier-Eng tasks require multi-step reasoning, numerical computation, and domain-specific knowledge to produce verifiable solutions. We curate 47 tasks (Phase 1) with normalized scoring and per-task leaderboards, enabling fine-grained comparison of model capabilities. Across all tasks, we observe consistent scaling laws: performance scales approximately as 1/t with compute per task and as 1/k with the number of independent attempts, suggesting that current frontier models are far from saturating engineering problem-solving.
Key Findings
score ∝ 1/t

Compute Scaling per Task

Performance on engineering tasks scales roughly as the reciprocal of thinking time per problem. Allocating more compute per task yields diminishing — but consistent — returns, with no saturation observed at current model scales.

score ∝ 1/k

Attempt Scaling (pass@k)

Best-of-k performance improves predictably with the number of independent attempts. The 1/k relationship holds across domains and model families, suggesting that diversity of reasoning paths is as important as raw capability.
