Performance Profile α

Cumulative performance profile Pm(α) as in the paper’s Evaluation protocol section (verbatim definitions below)

// Primary metric: average rank (paper, Evaluation protocol)

Source: Frontier-Eng paper arXiv:2604.12290, Evaluation protocol (“Primary metric: Average Rank”). For each task i and model m, let s⋆i,m be the best feasible score that model m obtains on task i within the benchmark interaction budget. All M models are ranked on that task by this score (higher is better): rank 1 to the best and rank M to the worst, with ties averaged. The average rank Rm across all N tasks is the mean of these per-task ranks; lower is better, and Rm = 1 means first on every task. The experiments use this as the primary cross-task comparison because it is unit-free and robust to heterogeneous task scales.

// Performance profile in α (paper, Evaluation protocol + experiments figure caption)

Source: same manuscript, Evaluation protocol (“Distributional analysis: Performance Profile”), adopting the framework of Dolan & Moré (2002) as cited there. For each task i, let s⋆best,i be the best score achieved by any model on that task. The performance ratio is ρi,m = s⋆best,i / s⋆i,m when s⋆i,m > 0, and +∞ otherwise. The performance profile is Pm(α) = (1/N) · |{ i : ρi,m ≤ α }| for α ≥ 1. The aggregate figure caption in the experiments states that α = 1.0 means attaining the task-best score (including ties), while α = 1.1 means the ratio between the task-best score and the model score is at most 1.1 (within 10% of best). The curves plotted here use the same definition on the frozen leaderboard task scores.
