Browser agent

Live page evidence plus reusable skills.

Core idea

Clone browser behavior at the skill level, not the click level.

The paper argues that raw browser traces are too long, noisy, and page-specific to reuse directly. Instead, successful trajectories can be distilled into transferable skill cards: procedural knowledge that captures when a strategy applies, what to observe, how to execute it, and how to recover when the page or task path diverges.

Abstraction

From events to procedures

Coordinates, selectors, and incidental page states are replaced with semantic goals, milestones, and terminal evidence.

Distribution

Many users, one library

Trajectory processing can run across shards, then merge local candidate skills into an auditable global library.

Execution

Retrieve, compose, ground

At test time, the browser agent retrieves relevant skills and grounds them into live page observations and actions.

Method

The trace-to-skill pipeline

BrowserBC lifts behavior cloning from one-step action imitation to graph-grounded skill distillation. The page below turns the method section into a compact flow that is easier to scan than the full PDF.

1

Clean trajectories

Normalize heterogeneous logs and remove interactions that do not change the task state.

2

Build graphs

Convert demonstrations into state-transition graphs with progress, verification, recovery, and failure edges.

3

Induce skills

Cluster reusable subgraphs into candidate procedures supported by multiple trajectory fragments.

4

Distill cards

Summarize each skill into intent, scope, observations, steps, evidence, failure modes, and provenance.

5

Retrieve at runtime

Rank skills by relevance, compatibility, graph connectivity, and historical reliability before execution.

Evaluation

Skill summaries improve WebArena-Hard success.

In the paper's WebArena experiment, the same browser agent is evaluated with skills off and with the BrowserBC skill library. The skill-on condition raises overall task success from 60.5% to 81.4%.

Skill card anatomy

What gets stored in the skill library?

A distilled skill is meant to be auditable and executable. It does not store benchmark answers; it stores procedure, evidence, and boundaries that help the agent choose actions on a live page.

Before acting

Intent and preconditions

  • What user goal the skill advances.
  • Which page state or domain it applies to.
  • What observations should be checked first.
During action

Procedure and grounding

  • High-level steps without brittle selectors.
  • Element-selection principles.
  • Milestones that indicate progress.
After action

Evidence and recovery

  • Terminal signs that the subtask is done.
  • Common failure modes.
  • Fallback paths when the page diverges.
The key design choice is confidence-aware use: the skill is a procedural prior, not a command to blindly follow. The paper notes that over-specific skills can occasionally regress tasks, so the agent should defer to live page evidence when it conflicts with the retrieved procedure.
